ChurchApps Ambassador Program

Master Implementation Plan - January 2026

Mission: Build a global network of local ambassadors who help churches adopt ChurchApps tools while maintaining quality, preventing abuse, and gathering valuable feedback.

Executive Summary

This plan outlines the complete system for launching the ChurchApps Ambassador Program, starting with Korea as a test case before opening to ~4 additional candidates and then scaling globally.

Key Principle: 95% automated, 5% personal touch. Every step should save time while building genuine relationships.

Core Requirements Met

  • Vetting: Multi-step application verifying Christian faith, references, and actual product usage
  • Monitoring: Email forwarding system to prevent scams/charging
  • Automation: ChurchApps forms, auto-responses, staged workflow
  • Personal Touch: Video call with Terry/Michael at key decision point
  • Proof Required: Photos of churches using ChurchApps products
  • Recognition: Website with ambassador map and achievement badges
  • Community: Private Facebook group for ambassadors

Phase 3: Process & Automation (Week 2)

G. 7-Step Ambassador Onboarding Process

Adapted from FaithTech, customized for ChurchApps

Step 1: Application Submitted (Day 0)
  • Auto-response: "Thank you, we'll review within 3 business days"
  • Notification to review team
  • Request photos via email if not provided
Step 2: Initial Review (Day 1-3)
  • Check: Is this person a Christian? (faith statement)
  • Check: Do they actually use ChurchApps? (photos, references)
  • Check: Any red flags? (scam indicators)
  • Decision: Proceed, Reject, or Request More Info
  • Email response with next steps or rejection
Step 3: Reference Check (Day 4-7)
  • Automated email to references
  • Simple form: "Do you know this person? Do you recommend them?"
  • Wait for at least 1 reference response
Step 4: Video Call with Terry/Michael (Day 8-10)
  • Calendly link sent for scheduling
  • 30-minute call to discuss vision
  • Personal connection and gut check
  • Assess readiness and commitment
Step 5: Final Decision (Day 11)
  • Team reviews call notes
  • Approve or decline
  • Email with decision
Step 6: Agreement Signing (Day 12-14)
  • Send DocuSign covenant
  • Wait for signature
  • Set up regional email group
  • Add to monitoring system
Step 7: Onboarding & Launch (Day 15+)
  • Welcome email with resource hub access
  • Add to private Facebook group
  • Add to website ambassador directory
  • Assign initial "Ambassador" badge
  • Send first 90-day goals template

Phase 5: Monitoring & Quality Control

Email Monitoring Protocol

  • Weekly review of ambassadors-monitor@churchapps.org
  • Flag any emails containing payment requests
  • Quarterly spot-check: random sampling of emails
  • Anonymous tip line for churches to report issues
  • Three-strike policy for violations

Quarterly Reporting System

  • Every 90 days, ambassadors submit report via form
  • Required fields: churches helped, meetings hosted, challenges faced, needs identified
  • Optional: photos, testimonials, translation work
  • Automated reminder 2 weeks before due
  • Missing 2 consecutive reports = probation status

Value Extraction

What we get from ambassadors:

  • Testimonials for marketing
  • Photos of churches using products (social media content)
  • Feature requests from local context
  • Translation assistance
  • Beta testing feedback
  • Case studies

Risk Mitigation

Risk: Ambassadors start charging churches

Mitigation: Email monitoring, clear covenant language, three-strike policy, public removal from directory

Risk: Low quality ambassadors who don't know products

Mitigation: Require photo proof of usage, reference checks, video call assessment, quarterly activity reports

Risk: Too many applications to handle

Mitigation: Limit to 1 ambassador per 500k population initially, automated rejection for incomplete apps

Risk: Ambassadors burn out

Mitigation: Only require 1 activity per quarter (low bar), peer support group, recognition system, graceful exit option

Risk: Theological issues/non-Christian trying to join

Mitigation: Faith statement required, reference checks with church leaders, video call

  • Ambassador grants program (small budget for events)

  • Automated badge awarding based on activity tracking
